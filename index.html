1. 프로젝트 소개

1.1 개발 배경

ADHD 성향이 있는 사람들은 기존 할일 관리 앱의 복잡한 인터페이스와 기능에 압도당하기 쉽습니다.

DayStep은 이러한 문제를 해결하기 위해 '지금 할 수 있는 가장 작은 것 하나'에 집중하는 철학을 바탕으로 설계되었습니다. 미룸 방지, 동기 부여, 관계 관리 기능을 통합하여 ADHD 성향 사용자도 쉽게 사용할 수 있는 할일 관리 경험을 제공합니다.

1.2 핵심 기능

  • 미룸방지: 프로젝트 기반 할일 관리, 진행률 시각화, 상태별 필터링
  • 머릿속 정리: 타임라인 뷰로 하루 일과 관리, 미완료 일정 추적
  • 실행 모드: "지금 할 수 있는 가장 작은 것 하나" - 포모도로 타이머 연동
  • 원동력: 개인 맞춤형 동기 부여 문구 관리
  • 관계 관리 CRM: 9가지 관심 유형, 양방향 부탁 추적, 연락 우선순위 추천, 통계 대시보드
  • AI 플래닝 어시스턴트: 4개 LLM 프로바이더 지원, Tool Use 오케스트레이션, ADHD 친화적 CAPS 메서드
  • 구독 결제: Paddle(웹) + RevenueCat(모바일) 통합 구독 관리
  • MCP 통합: ChatGPT Actions, Claude Code에서 DayStep 데이터 직접 관리

4. 머릿속 정리 - 실행

  • BUILD_TARGET 환경변수로 플랫폼별 빌드 분기 처리

3.2 상태 관리 전략

Zustand를 중심으로 클라이언트 상태를 관리하고, Immer로 불변성을 보장합니다. 오프라인 지원을 위해 Dexie.js(IndexedDB)를 사용하여 로컬 캐싱을 구현했습니다.

  • Zustand Store: 전역 UI 상태, 사용자 설정 관리
  • Supabase Realtime: 멀티 디바이스 실시간 동기화
  • Dexie.js: 오프라인 우선 데이터 접근, 온라인 복귀 시 자동 동기화

3.3 UX 최적화

  • 모바일 최적화 UI: React Modal Sheet을 활용한 바텀시트 네비게이션
  • 애니메이션: Framer Motion으로 부드러운 페이지 전환 및 마이크로 인터랙션
  • 드래그 앤 드롭: @dnd-kit으로 할일 순서 변경, 접근성(키보드) 지원
  • 다크 모드: next-themes로 시스템 설정 연동 및 수동 전환
  • 마케팅 랜딩페이지: 스크롤 기반 배경색 전환 (useScroll/useTransform), Smart CTA (플랫폼 감지), prefers-reduced-motion 접근성 지원

3.4 결제 아키텍처

웹과 모바일에서 동일한 구독 상태를 유지하기 위해 Paddle(웹)과 RevenueCat(모바일)를 통합했습니다.

  • Paddle Webhook: HMAC-SHA256 서명 검증, 구독 라이프사이클 관리 (553줄 Edge Function)
  • 크로스플랫폼 동기화: Paddle → RevenueCat API 연동으로 웹↔모바일 구독 상태 통합
  • 구독 이력: subscriptions, users, subscription_history 다중 테이블 동기화

3.5 AI 아키텍처

멀티 프로바이더 게이트웨이로 4개 LLM을 추상화하고, MCP 서버로 외부 AI 도구와 연동합니다.

  • AI Gateway: 프로바이더 팩토리 패턴으로 Claude/OpenAI/Gemini/Groq 통합 (850줄+)
  • Tool Use: 스트리밍/비스트리밍 지원, ADHD 플래닝 특화 시스템 프롬프트
  • MCP Server: JSON-RPC 2.0 기반 MCP 2024-11-05 스펙 구현 (1,300줄+)
  • ChatGPT 연동: OAuth 기반 ChatGPT Actions로 DayStep 데이터 직접 관리

3.6 트러블슈팅 & 문제 해결

하이브리드 앱과 멀티플랫폼 결제 시스템 구축 과정에서 만난 주요 기술적 문제와 해결 방안입니다.

1) Capacitor WebView OAuth 리다이렉트
  • 문제: WebView 환경에서 Google/Apple OAuth 콜백 URL이 앱으로 돌아오지 않음
  • 원인: 모바일 앱은 localhost 리다이렉트 불가, Custom URL Scheme 필요
  • 해결: Deep Link 설정 + Capacitor App Plugin으로 URL 가로채기 구현, 세션 토큰 브릿지 코드 작성
2) Paddle↔RevenueCat 크로스플랫폼 구독 동기화
  • 문제: 웹(Paddle)에서 결제한 사용자가 모바일에서 구독 상태 미인식
  • 원인: 두 결제 시스템이 독립적으로 동작, 구독 상태 공유 불가
  • 해결: Paddle Webhook → Supabase DB → RevenueCat API 연동 파이프라인 구축 (553줄 Edge Function)
3) BUILD_TARGET 환경 분리
  • 문제: Web/Mobile 빌드 시 Next.js SSR, export 모드 충돌
  • 원인: Next.js output 설정이 단일 값만 지원, 플랫폼별 분기 불가
  • 해결: 환경변수 기반 동적 next.config.ts 설정, cross-env로 플랫폼별 빌드 스크립트 분리
4) Optimistic Update + Realtime 충돌
  • 문제: 드래그앤드롭 후 Realtime 이벤트가 상태를 덮어쓰며 UI 깜빡임 발생
  • 원인: 낙관적 업데이트와 서버 Realtime 응답의 타이밍 불일치
  • 해결: 로컬 mutation ID 추적, 자신이 발생시킨 이벤트 필터링하여 중복 적용 방지
